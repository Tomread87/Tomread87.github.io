Introduction

The Website is a portal where my wife and I can share our most loved recipes.

Some of these recipes have been gathered through our years together and added in a little recipe book my wife cherishes very much. When I was asked in the module to create an actual website, I thought that it was the perfect occasion to bring all those recipes from the book to the web, so that we could share them with friends and family.

When designing the website, I wanted to have a simple structure with a navigation Menu at the top and the content underneath it. The website has a home page and then 4 pages categorized by meal type (starters, main dishes, everything that is part of baking and desserts).

I wanted to make sure that my wife and I could upload new recipes easily, so when designing the website, I created a template engine for the recipes. The recipes are found in a .js file called recipes.js and are saved as Objects. To add or remove a recipe it's sufficient to modify that .js file. When the webpage is loading it will load all the appropriate recipes from the file and the template engine in JavaScript will create a card for each recipe.

Recipes are loaded in each page as cards having a picture, a title, and a description of the recipe. If you click on the recipe card it will expand showing more in detail the ingredients and the steps to cook the recipe. All of the pictures come from freestock images websites or have been taken with a camera from me.

At the bottom of the web page there is a standard footer. I followed my wife's preferences and chose a palette of greys, coral and white colours.

Accessibility

When designing and developing my website I wanted to be sure that accessibility was taken into account. I installed a chrome extension called "Screen Reader" (it can be found here https://chrome.google.com/webstore/detail/screen-reader/kgejglhpjiefppelpmljglcjbhoiplfn?hl=en). https://chrome.google.com/webstore/detail/screen-reader/kgejglhpjiefppelpmljglcjbhoiplfn?hl=en).

This extension reads the content of a website and it's designed for people that are blind or have very little vision. I tried to navigate my website with my eyes closed using the extension. If I got stuck on the webpage I would change the design to make it more accessible.

Additionally I researched what makes a better alt text for images. I made sure that all recipe images have a good alt text.

Finally I added a feature that can change the website to dark mode. This mode makes the background dark and the text white so that people who have difficulties to read on white backgrounds can change nod to dark and have a better experience.

Usability

I made sure that the website is both mobile and pc friendly. I took in consideration and studied the responsiveness of the website. I made sure that all the media query would satisfy all possible, phones, tablets and PC screen sizes. The cards and content of the website changes design according to the width of the screen. Additionally on pc you have all the options to click, hover on links and cards while on mobile devices I added touch features on certain pages (such as swiping cards left or right in the dessert.html page).

I developed my website with an aesthetic and minimalist design. By choosing a predetermined colour palette and using the recognizable card design, users should not find difficulty in browsing and searching what they are looking for.

A third usability feature is a search bar at the top of each category. It can help user search for a particular recipe title or it can filter the recipe's by ingredients.
